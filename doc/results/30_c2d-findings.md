# C2D findings — wire `Request::Rename` + dispatch + `mv` shell + QEMU smoke

Phase **C2D** (`doc/plans/18_c2-detail.md`), the client surface over C2B's `Store::rename`
(PR #171) which itself sat on C2A's overlay re-key (PR #170). Branch `c2d-wire-rename-mv`,
based on `origin/main` @ `242933f`. Mechanical once C2B exists; no cas/Verus/spec/ledger
change.

## What landed

1. **`Request::Rename { handle, from, to }`** (`storage-server/src/lib.rs`). Appended **last**
   in the `Request` enum (after `ListTags`) so every prior postcard discriminant — and the
   committed `request_dispatch` corpus — stays stable (the B5 append-never-reorder discipline).
   It decodes to postcard discriminant `0x18` (24). `wire.rs` is unchanged: the header
   `[0x45,0x51,0x02]` does **not** bump, and the new variant rides the same serde/postcard path.

2. **The dispatch arm** (`Server::dispatch`). A verbatim mirror of `Write`/`Unlink`: `R_WRITE`
   gate via `lookup`, ref-only (`HandleTarget::Ref { name, subtree, .. }` else `ReadOnly`),
   `full_path(subtree, ·)` on **both** endpoints, `store.rename(name, from, to, now)`,
   `map_err(store_err)`, `Response::Ok`. The path-validation gate gained a `Rename` arm
   validating both paths (the single-path `|` chain can't carry two fields).

3. **`store_err`: `StoreError::NotFound => ErrorCode::BadPath`.** `rename` is the **sole**
   producer of `StoreError::NotFound` (`cas/src/store.rs:2135`, a missing source), so this maps
   nothing else. This resolves the deferral C2B's findings flagged (its `_ => Internal`
   catch-all was a placeholder "until C2D adds the proper wire arm").

4. **The `mv` shell built-in** (`user/shell/src/runtime.rs`). `mv <from> <to>` splits the arg on
   the first space, errors `usage: mv <from> <to>` if either side is empty, else
   `report(request(&Request::Rename { handle: root_handle(), from, to }))` — the `rm`/`write`
   pattern. `mv` added to the `help` line.

5. **A committed Rename fuzz seed** (`storage-server/fuzz/corpus/request_dispatch/rename`, 11
   bytes) generated by an `#[ignore]`d `gen_rename_corpus_seed` test mirroring cas's tag-3
   precedent. Both fuzzers already auto-cover the new variant — `request_dispatch` decodes raw
   bytes, `structured_request` drives the `#[derive(Arbitrary)]` — so no harness code changed.

- `kernel cargo build` green — the shell compiles with the `mv` arm.
- **QEMU smoke** (`scripts/run-demo.sh` under the CLAUDE.md process-group timeout harness):
  `[storaged] store mounted` → `serving`, then the witness —
  `write a x` → `ok`; `mv a b` → `ok`; `cat b` → `x`; `cat a` → `error: not found`;
  `ls` shows `b (1 bytes)` and **no `a`**; `mv a b` (now-missing source) → `error: bad path`
  (clean, no panic); `sync` → `ok` then `cat b` → `x` (the rename **flushes and commits
  durably**, exercising DD3's base-from-origin path end to end). No panic/`Corrupt`.

## Design resolutions worth recording

- **A failed `mv` must be `Response::Err`, never `Response::NotFound`.** The shell's `report()`
  has a catch-all `_ => out(b"ok\n")` (it only special-cases `Ok`/`SnapId`/`Err`), so returning
  `Response::NotFound` for a missing source would print **`ok`** — silently reporting a no-op
  rename as success. Mapping `StoreError::NotFound → ErrorCode::BadPath` routes it through the
  `Err` arm, which `err_name` renders as `error: bad path`. (`Read`/`Stat` legitimately return
  `Response::NotFound` because their callers, e.g. `cmd_cat`, decode it explicitly.)

- **Cross-subtree denial is structural — no extra check (DD4).** The arm `full_path`s both
  `from` and `to` under the handle's subtree, and `validate_path` rejects `..` up front, so a
  target outside the subtree is literally unnameable (rev1§4.9 "unnameable and therefore
  denied"). The dispatch test witnesses it: through a `pub` subtree handle, a `to` of
  `["..","escaped"]` is `BadPath`, while a legal `to` lands under `pub/`.

- **No `wire.rs` / `SB_VERSION` / `mkfs` change.** The wire op is pure postcard-append; the
  on-disk format and its v5 bump were C2B. C2D is wire + shell only.

## Negative control (anti-theater)

`rename_moves_a_file_and_denies_what_it_should` asserts the **moved content** (`cat b` ⇒ `x`,
`pub/moved` ⇒ `public info`) and the **disappearance** of the source (`Response::NotFound`),
not just `Response::Ok`. A no-op or wrong-name rename would read back `NotFound`/wrong data and
fail. It also pins the three denials: missing source ⇒ `BadPath`, `..` target ⇒ `BadPath`,
read-only handle ⇒ `Denied`.

## Out of scope (per the plan, unchanged)

A client-visible open/file-descriptor wire surface; cross-ref rename (a copy with new lineage);
the dirty-descendant directory-rename prefix-update optimization; C2C (unlink-while-open + the
open-handle interleaving proptest). The `ReadOnly` arm of the dispatch is an unreachable
backstop over the wire (a snapshot handle has write rights stripped at `OpenSnapshot`, so the
rights gate returns `Denied` first) — kept to mirror `Write`/`Unlink` exactly.
