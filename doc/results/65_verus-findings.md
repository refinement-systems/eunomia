## 1. The decision, and the protocol fact it realizes

The head-advance lives in `Store::commit` (the write path; ¬ß4.3 step 4). After
barrier 1 (chunks durable), the commit pops the contiguous prefix of WAL records
whose effects are already flushed into immutable tree, then reads the new
superblock `(wal_head, wal_next_seq)` off the first record that remains ‚Äî or
resets to the linear-WAL sentinel `(0, wal_seq)` + `wal_tail = 0` when the log
fully drains (¬ß4.4). This is the TLA+ `CommitPrepare.newHead`: *the longest
contiguous prefix of records whose effects are flushed.*

8b lifts that decision out of `commit` into a verified core, leaving the
surrounding I/O (the two fsync barriers, the chunk-store index frame, the new
superblock write) as plain-Rust callers ‚Äî the 7f/7g / 8a split (a verified core,
thin delegators).

**`advance_head(records: &[RecMeta], wal_seq: u64) -> HeadAdvance`** ‚Äî the verified
form of the `while let Some(rec) = self.wal_records.front() { ‚Ä¶ }` loop. `ensures`,
‚àÄ records:

```
r.n_flushed <= records.len()
forall|j| 0 <= j < r.n_flushed ==> records[j].flushed      // everything popped is flushed
r.n_flushed < records.len() ==> !records[r.n_flushed].flushed   // the head record is the first non-flushed one
r.n_flushed < records.len() ==>                                  // head/seq read off that record...
    r.head == records[r.n_flushed].off && r.next_seq == records[r.n_flushed].seq
r.n_flushed == records.len() ==> r.head == 0 && r.next_seq == wal_seq   // ...or the reset sentinel when all flushed
```

It is **total** ‚àÄ records and **terminating** (`decreases records.len() - i`). The
contract is the contiguous-flushed-prefix characterization: the popped prefix is
exactly the flushed records (clause 2), the head record is the first *un*flushed
one (clause 3) ‚Äî so the head never advances past an acked-but-uncommitted write,
which is the code-level shadow of the protocol's "replay recovers every unflushed
record." `HeadAdvance { n_flushed, head, next_seq }` returns the prefix length to
pop plus the new superblock fields.

## 2. The extraction ‚Äî `RecMeta` moves into the block, a no-`break` prefix scan

`RecMeta` (`{ seq, off, ref_name, flushed }`, crate-private, used only in
`store.rs`) **moves into the `verus!{}` block** so `advance_head` can name its
fields in the contract ‚Äî Verus must know the datatype to reason about
`records[j].flushed`. It erases to the same plain struct, so the rest of `store.rs`
(the `VecDeque<RecMeta>` field, the constructors in `mount`/`log_then_apply`, the
`for rec in &mut self.wal_records { rec.flushed = true }` in `flush_ref`) is
**unchanged** ‚Äî the `Vec<u8> ref_name` field rides along but the head-advance never
reasons about it (vstd `alloc` is on since 7g). Because `RecMeta` is private,
`advance_head` is **module-private** (no `pub` ‚Äî a `pub fn` taking a private type is
a Rust privacy error, and this lets the contract name `RecMeta`'s private fields
freely; cf. 8a's functions, which are `pub` because they take only scalars).

The body is the no-`break` short-circuit prefix scan (the `prolly.rs` `copy_range`
/ 7g opt-loop idiom; `broadcast use vstd::slice::group_slice_axioms` for the
exec‚Üîspec slice bridge):

```rust
let mut i: usize = 0;
while i < records.len() && records[i].flushed
    invariant
        i <= records@.len(),
        forall|j: int| #![trigger records@[j]] 0 <= j < i ==> records@[j].flushed,
    decreases records@.len() - i,
{ i += 1; }
if i < records.len() {
    HeadAdvance { n_flushed: i, head: records[i].off, next_seq: records[i].seq }
} else {
    HeadAdvance { n_flushed: i, head: 0, next_seq: wal_seq }
}
```

The short-circuit `&&` is what keeps `records[i].flushed` in bounds (only evaluated
when `i < records.len()`), and its loop-exit negation hands the `i < records.len()`
branch exactly `!records@[i].flushed` (clause 3) with no extra `assert` ‚Äî Verus
tracks the negated exec condition. `i += 1` is overflow-free (`i < len <=
usize::MAX`), discharged automatically. The reset branch (`i == records.len()`,
since `i <= len`) yields the sentinel; the other clauses hold vacuously there.

`commit`'s rewrite is the thin caller ‚Äî `make_contiguous()` (`VecDeque` ‚Üí
`&mut [RecMeta]`, auto-reborrowed to `&[_]`) gives the slice at zero copy, its
borrow ending when `advance_head` returns so the later `pop_front` loop is
borrow-clean:

```rust
let wal_seq = self.wal_seq;
let adv = advance_head(self.wal_records.make_contiguous(), wal_seq);
for _ in 0..adv.n_flushed { self.wal_records.pop_front(); }
let (wal_head, wal_next_seq) = (adv.head, adv.next_seq);
if self.wal_records.is_empty() { self.wal_tail = 0; }
```

`is_empty()` after popping `n_flushed` ‚ü∫ `adv.n_flushed == records.len()` ‚ü∫ the
original `front() == None` reset, so the behaviour is identical arm-for-arm ‚Äî which
the crash-injection proptests (running the rewritten `commit`) confirm.

  **49 verified, 0 errors**; the only friction was a low-confidence-trigger *note*
  (not an error) on the `forall|j| ‚Ä¶ records@[j].flushed` quantifier, silenced by

  obligation auto-gates. Verus stays pinned at `0.2026.06.07.cd03505`.

## 4. What stays the other tiers' / later sub-phases' job

- **Cross-commit head monotonicity (`new head >= old head`)** ‚Äî listed in the
  plan table ‚Äî is **deliberately not** in `advance_head`'s contract. It is a
  *Store-level* invariant: it rests on WAL offsets strictly increasing by
  construction in `log_then_apply` (each append bumps `wal_tail` by the record
  length), which the plain-Rust `commit` call site cannot hand Verus as a
  precondition (the `Store` is not Verus-tracked). Stating it would need a
  `sorted_offs` precondition unenforceable at that call site. The per-piece
  contract above is precondition-free and fully justifies the extraction; the
  monotone fact is left to the 8c/8d composition, where the surrounding invariant
  is in scope (the plan's explicit allowance: "per-piece contracts before the
  composed theorem", the ¬ß4.8 parallel of phase 6's deferred system clause).
- **The content-coverage half of `AckedWritesRecoverable`** ‚Äî "flushed ‚áí effects
  in the committed root", the last-write-wins semantics TLA+ abstracts to version
  numbers ‚Äî remains the `CommitProtocol` design gate; the crash-injection proptest
  stays the differential seam exercising the two halves composed against real tree
  content (master plan ¬ß4.8).

The TLA+‚Üîcode correspondence 8b lands is **`CommitPrepare.newHead ‚Üí advance_head`**
(adding to 8a's `LiveSlot ‚Üí pick_survivor` and the `Crash`-safety witness
`commit_target`). Phase 8c (`replay_bound`) extends the core to the replay-bound
decision (`Recover`); 8d composes `advance_head ‚àò replay_bound` into the

documentation-only closeout).
